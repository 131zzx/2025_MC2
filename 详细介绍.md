## 一、选题背景

MC2 设定在虚构岛国 Oceanus（海洋国）。该国长期以渔业为经济支柱，近年来旅游业快速扩张，经济结构发生显著变化。

政府设立了监督委员会 COOTEFOO（Commission on Overseeing the Economic Future of Oceanus，海洋国经济未来监督委员会），负责监测经济动向并给出政策建议。委员会成员由任命产生，没有直接立法或拨款权力，但在公务员体系中有较大影响力，并获得政府资金支持——理论上应保持中立。

近期 COOTEFOO 卷入一系列丑闻，舆论分裂为两派：

| 阵营  | 全称                                                       | 核心指控                                                     |
| :---- | :--------------------------------------------------------- | :----------------------------------------------------------- |
| FILAH | Fishing is Living and Heritage（渔业即生活与遗产）         | 委员会偏袒旅游业，忽视渔业这一历史支柱；部分成员对渔业有偏见 |
| TROUT | Tourism Raises OceanUs Together（旅游提升 Oceanus 共同体） | 委员会偏袒既得利益/渔业，不断「安抚」渔业，忽视旅游等新增长点；部分成员活在过去 |

记者 Edwina Darling Moray（E.D.） 在《Haacklee Herald》调查政府问责问题。她拿到了 TROUT 和 FILAH 各自使用的数据集（会议记录、差旅记录等），还获得了两方都没有的补充记录，并请人整理为统一格式的知识图谱。

你的角色：接替程序员，为 Moray 设计可视化与视觉分析工具。她不是图谱专家，但熟悉 Oceanus 本地情况——界面需要让她在上下文中探索指控，判断哪些成立、哪些无根据。

## 二、核心任务

每题需配适量图表，文字尽量简短。

### 问题 1：两方数据集是否「自证其说」？

基于 TROUT 和 FILAH 各自提供的数据，用视觉分析判断：

- 各方指控是否被其自身记录集支持？
- 能否可视化揭示 TROUT / FILAH 数据集中的采样偏见？
- COOTEFOO 成员行为在任一数据集中是否有偏见证据？

分析思路：不是先判委员会对错，而是先看「各方拿出的证据本身是否偏颇」——例如 FILAH 数据是否过度呈现渔业议题、TROUT 数据是否过度呈现旅游议题。

### 问题 2：全量数据下委员会整体是否偏袒？

Moray 将 TROUT、FILAH 数据与补充记录合并为完整知识图谱（`journalist.json`）。需要：

- 可视化 COOTEFOO 成员如何分配时间（会议、差旅、议题参与）
- 判断委员会整体是否偏袒某一方（渔业 vs 旅游）
- 用视觉证据支撑结论

### 问题 3：不完整数据 vs 全量数据的结论对比

TROUT 和 FILAH 数据集均不完整。需要对比：

- 分别基于 TROUT、FILAH 得出的结论
- 与全量数据集中实际行为的差异
- TROUT 的指控在全量背景下是被加强、削弱还是不变？

### 问题 4：人物级对比分析

设计可视化，让 Moray 选择一名委员会成员，对比该人在不同数据集中的行为差异，突出「各数据集讲的故事有何不同」。子问题包括：

1. 至少选一名被 TROUT 指控的成员，展示用更完整数据后对其活动的理解如何变化
2. 原 TROUT 数据中最关键的缺失证据是什么？
3. 在全量背景下，FILAH 采样偏见对谁的行为影响最大？
4. 在全量背景下，展示 FILAH 数据的偏见。

## 四、知识图谱结构详解

### 4.1 节点类型（9 类）

| 类型                  | 含义                               | 典型属性                                                    |
| :-------------------- | :--------------------------------- | :---------------------------------------------------------- |
| `entity.person`       | COOTEFOO 委员会成员                | `name`, `role`                                              |
| `entity.organization` | 相关组织（渔业公司、票务、协会等） | `id` 即为对应组织名                                         |
| `topic`               | 经济议题/项目                      | `short_topic`, `long_topic`                                 |
| `meeting`             | 委员会会议                         | `date`（如 "Meeting 1" 或 "07-03-40"）                      |
| `discussion`          | 会议中的讨论项                     | `short_title`, `long_title`                                 |
| `plan`                | 计划/行动                          | `plan_type`（Travel / Report / proposal / presentation 等） |
| `place`               | 地点                               | `lat`, `lon`, `zone`, `zone_detail`, `name`                 |
| `trip`                | 实际差旅行程                       | `date`, `start`, `end`                                      |
| 无 `type` 的节点      | 部分 place 节点缺省 type           | 仍可通过坐标等属性识别                                      |

### 4.4 边（关系）类型

图谱中的 `links` 通过 `role` 字段表达语义（部分边无 `role`，见下文）：

| role          | 含义                  | 示例                                                         |
| :------------ | :-------------------- | :----------------------------------------------------------- |
| `part_of`     | 讨论/计划属于某次会议 | Meeting_1 → fish_vacuum 讨论                                 |
| `about`       | 讨论/计划关于某议题   | 讨论 → expanding_tourist_wharf                               |
| `plan`        | 计划关联议题          | 差旅计划 → topic                                             |
| `travel`      | 计划关联目的地        | 差旅计划 → place（地点 ID）                                  |
| `participant` | 参与者及立场          | 含 `sentiment`（-1~1）、`reason`、`industry`（fishing/tourism） |
| （无 role）   | 行程关联              | trip → 人员；trip → 地点（含 `time` 时间戳）                 |

participant 边尤其重要——包含情感倾向与行业标签，可直接用于「偏渔业还是偏旅游」的量化分析。例如：

{

  "role": "participant",

  "sentiment": 0.5,

  "reason": "Supports tourism growth",

  "industry": ["tourism"],

  "source": "expanding_tourist_wharf_Travel_Harbor_Route_Solutions",

  "target": "Simone Kat"

}

### 4.5 数据层次关系（逻辑链）

Meeting（会议）

  └─ part_of ─→ Discussion / Plan（讨论 / 计划）

​        ├─ about ─→ Topic（议题）

​        ├─ plan ─→ Topic

​        ├─ travel ─→ Place（目的地）

​        └─ participant ─→ Person / Organization（参与者及立场）

Trip（实际行程）

  ├─ → Person（出行人，无 role）

  └─ time + → Place（途经/到达地点）

议题命名有规律：`{topic}_{Meeting_N}_{动作}` 或 `{topic}_Travel_{地点名}`，便于从 ID 前缀提取 topic。

### 4.6 地理与路网辅助数据

- place 节点：含 `lat`/`lon`、`zone`（commercial / residential / industrial / tourism / government）
- oceanus_map.geojson：岛屿多边形（如 Suna Island），可用于底图
- road_map.json：道路节点与连边，可用于 trip 路径还原、时空轨迹可视化

## 五、三个数据集的关键差异

| 维度       | FILAH       | TROUT | journalist（全量） |
| :--------- | :---------- | :---- | :----------------- |
| 节点数     | 396         | 164   | 740                |
| 边数       | 765         | 378   | 2436               |
| 委员会成员 | 3 人        | 6 人  | 6 人               |
| trip 节点  | 189（最多） | 18    | 342                |
| meeting    | 12          | 13    | 16                 |
| topic      | 14          | 14    | 15                 |

初步解读：

- FILAH 数据量大但人员不全，且 trip 记录极多——可能过度强调「出行/渔业相关活动」叙事
- TROUT 数据量最小，节点数不到 FILAH 的一半——可能遗漏大量会议与行程
- journalist 含额外 meeting（如 Meeting_13/15 带真实日期 `07-03-40`）、更多 trip 与 1 个额外 topic——这些是「缺失证据」分析的重点

## 六、建议的可视化方向

针对 4 大问题，常见且有效的视觉分析路径：

问题 1 & 4（数据集偏见）

- 议题分布堆叠条形图 / 桑基图（渔业 vs 旅游占比）
- 各数据集「人物-议题-活动」覆盖矩阵（谁被记录、谁被遗漏）
- 平行坐标 / 小 multiples：同一人在 FILAH vs TROUT vs 全量下的活动对比

问题 2（委员会整体时间分配）

- 时间轴：会议、差旅、议题参与随 Meeting/日期变化
- 地理地图：place + trip 轨迹，按 zone/industry 着色
- 成员雷达图 / 热力图：每人参与渔业/旅游议题比例

问题 3（结论对比）

- 三联对比仪表盘：TROUT 结论 | FILAH 结论 | 全量真相
- 差异高亮：全量有而 TROUT/FILAH 缺失的节点/边

问题 4（人物 drill-down）

- 可选人物 + 三数据集联动的 ego-network 或 timeline
- 缺失证据清单：全量中存在、TROUT 中不存在的 plan/trip/participant 边
